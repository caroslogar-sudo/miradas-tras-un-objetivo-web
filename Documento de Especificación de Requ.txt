Documento de Especificación de Requisitos (ERS)Proyecto: Web Fotográfica de Semana Santa Nacional1. Visión GeneralEl proyecto consiste en una plataforma web para la exposición y venta personalizada de fotografías personales de la Semana Santa nacional. El enfoque principal es el valor artístico de las imágenes, permitiendo una interacción comunitaria centrada exclusivamente en la fotografía y facilitando un canal de venta directa a través de correo electrónico.2. Objetivos del SistemaExhibición: Mostrar imágenes de alta calidad con metadatos específicos (hermandad, ciudad, año).Interacción Moderada: Permitir comentarios que se ciñan únicamente a la técnica o contenido de la obra.Comercialización Directa: Facilitar la compra de copias físicas o digitales mediante contacto por email, sin pasarela de pago interna.Gestión con Supabase: Utilizar un backend escalable para datos, imágenes y autenticación.3. Requisitos Funcionales (RF)IDRequisitoDescripciónRF-01Carga de ContenidoEl administrador podrá subir fotos, añadir títulos, descripciones y etiquetas (tags).RF-02Galería DinámicaVisualización en rejilla (grid) con filtros por ciudad o temática y visualización individual (lightbox).RF-03Sistema de ComentariosSección de comentarios por foto con un aviso legal visible sobre la temática permitida.RF-04Contacto para VentaBotón "Interesado en esta fotografía" que activa un enlace mailto con los datos de la foto.RF-05Panel de ModeraciónCapacidad para el administrador de eliminar comentarios que incumplan las normas.RF-06Seguridad de ArchivosImplementación de marcas de agua automáticas en la vista previa para proteger la autoría.4. Especificación Técnica (Supabase)Para la persistencia y gestión, se utilizarán los siguientes servicios de Supabase:4.1. Base de Datos (PostgreSQL)Se requieren dos tablas principales:fotografias: id (UUID), titulo (text), descripcion (text), url_storage (text), localidad (text), fecha_creacion (timestamp).comentarios: id (UUID), foto_id (FK), nombre_usuario (text), contenido (text), aprobado (bool), fecha (timestamp).4.2. Storage (Almacenamiento de Imágenes)Bucket "Galeria": Almacenará los archivos originales y optimizados. Se aplicarán políticas RLS (Row Level Security) para que solo el admin tenga permisos de escritura (INSERT/DELETE).4.3. AutenticaciónUso de Supabase Auth para el acceso del administrador al panel de gestión de contenido.5. Requisitos No Funcionales (RNF)Rendimiento: Uso de Lazy Loading para asegurar que la carga de imágenes de alta resolución no afecte la velocidad inicial de la web.Disponibilidad: La web debe ser Responsive (adaptable a móviles y tablets).Interfaz (UI): Diseño sobrio, con predominio de tonos oscuros o neutros para resaltar el color y la luz de las fotos de Semana Santa.6. Flujo de Venta DetalladoPara evitar la complejidad de una tienda física de pago:El usuario selecciona una foto y pulsa "Contactar para comprar".La web genera un correo electrónico automático dirigido a tu dirección con el asunto: Consulta de compra: [Título de la Foto].El proceso de pago y envío se acuerda externamente (transferencia, Bizum, etc.), cumpliendo así con tu requisito de no tener tienda de pago integrada.
